Problem 005


Given a multiset of integers, return whether it can be partitioned into two subsets whose sums are the same.



For example, given the multiset {15, 5, 20, 10, 35, 15, 10}, it would return true, since we can split it up into {15, 5, 10, 15, 10} and {20, 35}, which both add up to 55.

Given the multiset {15, 5, 20, 10, 35}, it would return false, since we can't split it up into two subsets that add up to the same sum.



Solution



The above problem can be solved using a technique called as Meet in the middle approach.

I would recommend watching this video to get an overview of the MitM technique. https://youtu.be/57SUNQL4JFA

We need to partition the set in such a way that the 2 parts have the same sum. That means that the problem can be simplified to finding a set which has the sum equal to total_sum/2.

The brute force approach would generate all the combinations of N (size of the set) and pick the 2 sets that have the sum total_sum/2.

Time complexity, in this case, would be O(2^N).

Using meet in the middle technique, we can reduce the time complexity to be O(2^(N/2)), which for big values of N would be a huge improvement.

Overview of the algorithm looks as follows:
1. Divide the array into 2 parts, A and B.
2. Generate all combinations and their sum for the set A and store that in a map. This can be done in O(2^(N/2))
3. Generate all the combinations for the set B.
4. For each combination sum K for the set B, check if the sum (total_sum/2) - K is present in the map.
5 If yes, then we have found the solution.
