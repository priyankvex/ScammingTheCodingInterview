Problem 011


Implement the LFU cache with constant time operations for get, set and delete


Implement an LFU (Least Frequently Used) cache with the following operations supported:

  • set(key, value): sets key to value. If there are already n items in the cache and we are adding a new item, then it should also remove the least frequently used item. If there is a tie, then the least recently used key should be removed.
  • get(key): gets the value at key. If no such key exists, return null.
  • delete(key): deletes a key for the cache.

Each operation should run in O(1) time.



Solution



In least frequently used cache, when the cache becomes full, we remove the item that has the least number of usage frequency.
LFU cache has many applications where a LRU cache won't give the required performance optimizations.
Ex. In DNS caching in the routers, we don't want the most popular DNS cache to be removed when a new entry is inserted.

If we were allowed to have an O(log N) complexity for the operations, we could've used the heap approach. But we have to provide all the operations in constant time.

Overview of the algorithm:
The idea is to use some special data structures to make this possible.
1. A list of frequency bucket is used to maintain all the frequency count for the items in the cache.
2. Each item belongs to the items list of one of the frequency node which represents its usage frequency.
3. Whenever the cache overflows, we remove an item by going to the items list of the first frequency node (as it'll have the minimum usage frequency).
4. When an item is accessed, we move it to the items list of the one bigger frequency count node.

All these operations can be done in constant time.
