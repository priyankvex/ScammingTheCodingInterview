Problem 009


Given a 2D matrix of characters and a target word, write a function that returns whether the word can be found in the matrix by going left-to-right, or up-to-down.



For example, given the following matrix:

[['F', 'A', 'C', 'I'],
['O', 'B', 'Q', 'P'],
['A', 'N', 'O', 'B'],
['M', 'A', 'S', 'S']]

The target word 'FOAM', you should return true, since it's the leftmost column. Similarly, given the target word 'MASS', you should return true, since it's the last row.



Solution



There are a bunch of ways and optimizations that we can do to solve the problem.

Brute Force:

One approach that doesn't use any extra space is to search for the first character of the target word in the matrix.
For each occurrence found, check the row and column for the word.

If we have to make multiple searches, each search needs an O(N*M) to search the first character in the matrix.


First Optimization:

One optimization can be to pre-process the matrix and create a map of the characters against the list of their occurrences.

This way, we can directly start with the occurrences of the first character and avoid the matrix traversal for each search query.
This does require extra space though.

This is the approach that we'll with here.


More Optimizations:

Few more optimizations can be possible based on early disqualification of the first letter occurrences.
For example, we can check for the last character too and check if there are any first character occurrences which don't have any last character occurrences in the same row or column.
Another criterion can be to include the length of the word too.
