Problem 003


Implement a URL Shortner



Implement a URL shortner with following functions:

1. Shorten URL: Given a URL, returns a shortened URL.

2. Restore URL: Given a shortened URL, returns the original URL.

Few things to consider:
 1. What if we enter the same URL twice? Will we want the same shortened URL or a different one.



Solution



A very basic approach can be, to hash the URL and take the first or last  6 characters from it.
Though this approach seems easy, it has a couple of pitfalls:

1. Hashes are not meant to be sliced, this can result in a very high collision rate.

2. What if we enter the same URL twice? Being a good service, we'll want it to give a different URL for each time.


There are many solutions possible to this problem. I would recommend watching this video too,
https://www.youtube.com/watch?v=fMZMm_0ZhK4.


A possible solution to this problem can be as follows:

1. Insert URL in the database to get an ID.
2. This ID will be unique to each URL generated.
3. Suppose our URL can be at maximum, 6 characters long, and we have characters from [a-z], [A-Z] and [0-9] i.e of 62 characters.
4. We can convert the database ID, to a base 62 number.
5. This base 62 number will be the shortened URL.
6. To get the original URL, we convert the base 62 number to the base 10 number.
7. This base 10 number will give us the database ID of the original URL, using which we can fetch the original URL.


Let's see take an example:

ID = 123456789
Shortened URL = "iwaUH"


Even this solution has a pitfall, that is that the URLs generated are sequential. 
One way to handle that can be to first hash the URL and assign it a bucket, each bucket can maintain its own counter for the IDs.
(We'll skip implementing this here.)
