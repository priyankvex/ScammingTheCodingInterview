Problem 021


Generate all possible words for given numbers of a keypad


Given a mapping of digits to letters (as in a phone number), and a digit string, return all possible wprds the number could represent. You can assume each valid number in the mapping is a single digit.

For example if:
{“2”: [“a”, “b”, “c”], 3: [“d”, “e”, “f”], …}
then “23” should return
[“ad”, “ae”, “af”, “bd”, “be”, “bf”, “cd”, “ce”, “cf"].



Solution



This is a classic brute-force problem.

We can start with taking the characters that the first number represents and for each of those characters consider all the letters that the next number represents.

We stop when we have reached the end.

The time complexity of this solution is O(4^N) where N is the number of digits.
